# MAT-1284 — mistakes

Newest first.

## Verified against a stale copy of the script (headless capture showed empty fuzzel)
- **What:** During the headless screenshot capture I `scp`'d `link-search.py` to the laptop, then later added a `LINK_SEARCH_DUMP` debug mode to the source but re-ran the capture without re-copying — so `python3 /tmp/link-search.py` dumped 0 lines and the first screenshot was of an empty launcher.
- **Fix:** Re-`scp` the updated script before re-running; confirmed `LINK_SEARCH_DUMP=1 … | wc -l` = 86630 before capturing.
- **Lesson:** When verification runs a copied artifact on a remote host, re-sync the artifact after every source edit — or verify a checksum/feature probe first.

## `nix eval .#…` didn't see new files until `git add`
- **What:** Evaluating the laptop config errored with `path '…/link-search/default.nix' does not exist` even though the files were on disk.
- **Fix:** git-based flakes only see tracked/staged files — `git add` the new paths (path-scoped), then eval.
- **Lesson:** Stage new flake files before any `nix eval`/`nix build` on a git flake.
